# 开发目录

## 1. 项目概览

- 技术栈：React 19 + TypeScript + Vite + Zustand + Tailwind CSS
- 入口文件：`src/main.tsx`
- 根组件：`src/App.tsx`
- 全局状态：`src/store/gameStore.ts`
- 统一数据配置：`src/data/*`

项目当前是一个“手机壳内的多系统经营游戏”，围绕聊天、赚钱、送礼、关系变化和战绩统计展开。

## 2. 已实现功能

### 2.1 聊天系统

文件：

- `src/systems/chat/ChatList.tsx`
- `src/systems/chat/ChatRoom.tsx`
- `src/systems/chat/GiftPicker.tsx`
- `src/systems/chat/chatEngine.ts`

能力：

- 展示所有角色会话列表，按最近消息排序
- 显示未读数量、好感度、关系阶段、角色状态
- 进入单聊后发送文本消息
- 支持快捷回复按钮
- 支持在聊天中送礼
- 显示 AI 回复来源：云端 AI / 本地保底
- 打工期间禁止聊天和送礼
- 被拉黑后禁止继续互动

### 2.2 角色与好感度系统

文件：

- `src/systems/girls/girlProfiles.ts`
- `src/systems/girls/affectionLogic.ts`
- `src/systems/girls/promptBuilder.ts`
- `src/data/girls/*.json`
- `src/data/balance.json`

能力：

- 自动扫描 `src/data/girls/*.json` 生成角色列表
- 每个角色包含基础资料、人设、礼物偏好、快捷回复、AI prompt、保底回复策略
- 好感度范围 0-100
- 关系阶段由 `balance.json -> affectionTiers` 控制
- 角色状态包括：`normal` / `suspicious` / `angry` / `blocked`
- 打工失联会扣好感，并给每个角色追加一条抱怨消息
- 礼物偏好会影响好感增减

### 2.3 打工系统

文件：

- `src/systems/earning/EarningPage.tsx`
- `src/systems/earning/jobs.ts`
- `src/systems/earning/DeliveryGame.tsx`
- `src/systems/earning/ReviewGame.tsx`
- `src/systems/earning/deliveryMap.ts`
- `src/data/economy/jobs.json`

当前工作类型：

- `送外卖`
  - 交互型工作
  - 进入随机地图窗口
  - 使用十字键按钮或键盘方向键控制骑手移动
  - 分为“取外卖”和“送外卖”两段
  - 到达目标点后完成订单并结算
- `淘宝刷单`
  - 交互型工作
  - 将三排星级全部点满后发布评价
  - 每完成一单奖励 1 元
  - 无次数限制
- `写代码`
  - 计时型工作
  - 开工后倒计时，结束自动结算
- `当主播`
  - 计时型工作
  - 奖励支持随机区间

实现说明：

- 工作配置支持两种模式：
  - `completionMode: "timer"`：倒计时自动结算
  - `completionMode: "manual"`：完成交互后手动结算
- 玩家处于打工状态时，`timeStatus = "working"`
- 计时型工作使用 `App.tsx` 内的轮询自动触发 `finishWork`
- 交互型工作完成后，由具体页面调用 `finishWork`

## 3. 核心数据入口

### 3.1 `src/data/girls/*.json`

角色数据来源。每个角色文件包括：

- 基础资料
- 说话风格
- 兴趣点
- 开场白
- 快捷回复
- 礼物偏好
- 打工抱怨模板
- AI prompt 分层配置
- 本地保底回复策略

### 3.2 `src/data/economy/gifts.json`

礼物配置来源。字段包括：

- `id`
- `name`
- `emoji`
- `price`
- `description`
- `advice`

### 3.3 `src/data/economy/jobs.json`

工作配置来源。字段包括：

- `id`
- `name`
- `emoji`
- `duration`
- `reward`
- `description`
- `rewardRange`
- `mode`
- `completionMode`

### 3.4 `src/data/balance.json`

全局数值来源。包括：

- 开局资金
- 初始好感
- 好感阶段
- 状态阈值
- 礼物增减数值
- 打工失联惩罚
- 计分公式

### 3.5 `src/data/ui-strings.json`

统一界面文案来源。所有新页面的按钮、提示、标题都应该加在这里，不要散写到组件里。

## 4. 主要状态与接口

### 4.1 类型定义

文件：

- `src/store/gameTypes.ts`

核心接口：

- `Gift`
- `Message`
- `GirlProfile`
- `GirlState`
- `JobDefinition`
- `CurrentJob`
- `GameState`
- `ActionResult`

其中：

- `JobDefinition` 定义工作配置
- `CurrentJob` 定义当前进行中的工作实例
- `CurrentJob.completionMode` 用于区分自动结算和手动结算

### 4.2 Store Action 接口

文件：

- `src/store/gameStore.ts`

当前对外动作包括：

- `sendMessage(girlId, text)`
- `sendGiftInChat(girlId, giftId)`
- `getChatHistory(girlId)`
- `getUnreadCount(girlId)`
- `markGirlRead(girlId)`
- `buyGift(giftId)`
- `getInventory()`
- `useGift(giftId)`
- `canAfford(giftId)`
- `getShopItems()`
- `startWork(jobType)`
- `finishWork()`
- `isPlayerBusy()`
- `getWorkProgress()`
- `updateAffection(girlId, change, mood?)`
- `resetGame()`

状态变化要点：

- `startWork()` 会把 `player.timeStatus` 切到 `working`
- `finishWork()` 会发工资、累计收入、清空当前工作，并统一计算打工期间的掉好感逻辑
- 所有关键状态变化后会重新计算 `score`、`gameOver` 和 `gameOverReason`

## 5. AI 与回复链路

文件：

- `src/config/api.ts`
- `src/systems/chat/chatEngine.ts`
- `src/systems/girls/promptBuilder.ts`
- `src/data/prompts/meta.txt`

流程：

1. `ChatRoom` 调用 store 的 `sendMessage`
2. store 内部调用 `generateChatReply`
3. 若已配置 SiliconFlow API，则请求云端模型
4. 若云端不可用，则回退到本地规则回复
5. 返回 `reply / affectionChange / mood / source / debugReason`
6. store 更新消息列表、好感度、情绪、状态和调试信息

当前外部接口配置：

- `VITE_SILICONFLOW_BASE_URL`
- `VITE_SILICONFLOW_API_KEY`
- `VITE_SILICONFLOW_MODEL`

## 7. 后续扩展建议

- 如果继续加新工作，优先复用 `jobs.json + JobDefinition + completionMode`
- 如果是新的交互型工作，建议在 `src/systems/earning/` 下单独建组件
- 所有新文案继续写入 `src/data/ui-strings.json`
- 如果要做随机事件，可从 `src/systems/events/eventEngine.ts` 开始接入
- 如果要新增角色，只需要新增 `src/data/girls/*.json`
